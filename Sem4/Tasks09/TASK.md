# Семестр 4 — Задание 9. Мини-СУБД: клиент-сервер по TCP (v7)

Развитие [Задания 8](../Tasks08/TASK.md): та же СУБД (список + структуры быстрого поиска по `name`/`phone`/`group`), но разделённая на сетевые клиент и сервер по TCP.

## Требования к программам

1. Структуры данных, команды, `config.txt` — как в [Задании 8](../Tasks08/TASK.md).
2. Программа должна получать все параметры в качестве аргументов командной строки и стандартного ввода.
3. Программа состоит из **двух исполняемых файлов**:
   - `client` — читает команды со стандартного ввода, отправляет их серверу по TCP, принимает результаты и выводит их (если требуется) в стандартный вывод. Аргументы: `host` – IP-адрес сервера, `port` – номер порта. Например: `cat commands.txt | ./client 127.0.0.1 5555 > result.txt`.
   - `server` — выполняет команды, приходящие по сети от клиентов. Аргументы: `filename` – файл со списком, `port` – номер порта для приёма соединений. Например: `./server a.txt 5555`.
4. Класс "список" должен содержать функцию ввода из файла (формат — как в [Задании 3](../Tasks03/TASK.md): `Слово-i Целое-число-1 Целое-число-2`, все записи различны).
5. Вывод результата в `main`:

   ```cpp
   printf ("%s : Result = %d Elapsed = %.2f\n", argv[0], res, t);
   ```

6. Для справки в задании приведён пример простейшего TCP клиента и сервера на сокетах Berkeley (`socket`, `bind`, `select`/`accept`, обмен строками) — `ser_tcp.c` (сервер: принимает соединения, переводит полученную строку в верхний регистр и отправляет обратно) и `cli_tcp.c` (клиент: подключается по DNS-имени/IP и порту, отправляет строку, читает ответ). Реализация в задании должна использовать тот же базовый механизм TCP-сокетов для передачи команд СУБД и результатов вместо учебного примера с одной строкой.

## Задачи

Реализовать те же **две структуры поиска по `name`/`phone`** и **индекс по `group`**, что и в [Задании 8](../Tasks08/TASK.md) (список 38 вариантов — там же), но в архитектуре клиент-сервер: `server` хранит базу и обрабатывает запросы, `client` — тонкий клиент, пересылающий команды и печатающий ответы.
