# Семестр 4 — Задание 8. Мини-СУБД: индекс по группе поверх двух структур (v6)

Развитие [Задания 7](../Tasks07/TASK.md): добавляется третья структура поиска — по полю `group`, которое (в отличие от `name`/`phone`) имеет очень много совпадающих ключей.

## Требования к программам

Те же, что в [Задании 6](../Tasks06/TASK.md) (список + структуры быстрого поиска, команды, `config.txt`).

## Задачи

- В [Задании 7](../Tasks07/TASK.md) уже реализованы две структуры быстрого поиска по `name` и `phone`, рассчитанные на малое количество совпадающих ключей.
- В этом задании добавляется структура поиска по полю `group` (много совпадающих ключей): **упорядоченный по `group` массив длины 1000**, в каждом узле которого (при известном номере группы) создаётся структура поиска, реализованная в предыдущих заданиях — по одной на каждое встретившееся значение `group`. Если номер группы неизвестен, поиск идёт как раньше — через структуры из [Задания 7](../Tasks07/TASK.md) по всей базе.
- Список из 38 доступных структур поиска — тот же, что в [Задании 6](../Tasks06/TASK.md).
