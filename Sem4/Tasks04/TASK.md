# Семестр 4 — Задание 4. Мини-СУБД: составные условия и выбор полей (v2)

Развитие [Задания 3](../Tasks03/TASK.md): условия `and`/`or`, произвольный список выводимых полей, `not like`.

## Требования к программам

1. Дополнительно к структурам из [Задания 3](../Tasks03/TASK.md):
   - `condition` пополняется значением `nlike` (строки, не соответствующие образцу).
   - Новый enum `ordering` — какие поля выводить: `none, name, phone, group`.
   - `record::print` теперь принимает массив `ordering order[]` (по умолчанию — `{name, phone, group}`), задающий порядок вывода полей.
   - Новый enum `operation` — логическая связка условий: `none, land, lor`.
   - `command` дополняется полем `operation op` (условия применяются к разным полям одной записи, связанные `and`/`or`).
2. Задача программы — как в [Задании 3](../Tasks03/TASK.md), но команда имеет вид `<условия на выводимые поля> where <условие поиска>`.
3. `<условия на выводимые поля>`: список полей через запятую без повторений (например `group, name`), либо `*` (эквивалент `name, phone, group`).
4. `<условие поиска>`: одно, два или три условия на **разные** поля `record`, связанные одним и тем же оператором (`and` либо `or`) — `<условие1>`, `<условие1> and <условие2>`, `<условие1> or <условие2>`, и т.д. до трёх условий.
5. `<условие поиска на одно поле>` — как в [Задании 3](../Tasks03/TASK.md), плюс `<поле> not like <образец>` (условие выполнено, если поле **не** соответствует образцу).
6. Примеры команд:
   - `group, name where phone = 1234567 and name = Student`
   - `* where phone >= 1234567 and name like St%`
   - `name where name not like St% and phone = 1234567 and group = 209`
7. Аргументы командной строки и формат вывода в `main` — как в [Задании 3](../Tasks03/TASK.md).

## Задачи

Написать программу мини-СУБД (список `record`, команды со стандартного ввода). Задание оценивается по качеству системы классов и по реализации:

1. Условие `<поле> not like <образец>` со всеми спецсимволами.
2. Задание `<условия на выводимые поля>` в виде `*`.
3. Задание `<условия на выводимые поля>` в виде списка полей.
4. Условие `and`.
5. Условие `or`.
6. Поддержка всех перечисленных условий в едином потоке запросов.
