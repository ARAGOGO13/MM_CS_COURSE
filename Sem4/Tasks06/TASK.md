# Семестр 4 — Задание 6. Мини-СУБД: структура быстрого поиска (v4)

Развитие [Задания 5](../Tasks05/TASK.md): класс "База данных", хранящий записи в двунаправленном списке (как раньше) плюс дополнительную структуру для быстрого поиска по одному полю.

## Требования к программам

1. Структуры данных, команды (`insert`/`select`/`delete`/`quit`), грамматика условий и сортировки — те же, что и в [Задании 5](../Tasks05/TASK.md).
2. Задача программы:
   - Построить класс "База данных", содержащий контейнер записей (двунаправленный список) **и** структуру для быстрого поиска.
   - Считать список из файла, затем обрабатывать команды со стандартного ввода, используя структуру быстрого поиска там, где это возможно, и линейный просмотр списка — там, где нет.
3. Аргументы командной строки и формат вывода в `main` — как в [Задании 3](../Tasks03/TASK.md).
4. Дополнительные параметры структуры поиска (`k`, `m` — если есть в алгоритме) читаются из файла с фиксированным именем `config.txt` в каталоге исполняемого файла: строки, начинающиеся с `#`, и пустые строки игнорируются; параметры — целые числа, разделённые пробелом/`\t`/`\n`.

   ```cpp
   #include <libgen.h>
   const char * config_name = "config.txt";
   unique_ptr<char []> exe_path = make_unique<char []> (strlen (argv[0]) + 1);
   strcpy (exe_path.get (), argv[0]);          // dirname изменяет аргумент — работаем с копией
   char *dir = dirname (exe_path.get ());      // каталог исполняемого файла
   // config_path = dir + "/" + config_name
   ```

## Задачи

Требуется реализовать **одну** из следующих структур для быстрого поиска объектов по полю `name` или `phone`:

1. Динамический вектор векторов длины `k`, упорядоченных по имени.
2. Динамический вектор векторов длины `k`, упорядоченных по номеру телефона.
3. B-дерево по имени, основание `m`.
4. B-дерево по номеру телефона, основание `m`.
5. AVL-дерево поиска по имени.
6. AVL-дерево поиска по номеру телефона.
7. Красно-чёрное дерево поиска по имени.
8. Красно-чёрное дерево поиска по номеру телефона.
9. Хеш по первым `k` буквам имени, на базе массива списков.
10. Хеш по первым `k` цифрам телефона, на базе массива списков.
11. Хеш по последним `k` буквам имени, на базе массива списков.
12. Хеш по последним `k` цифрам телефона, на базе массива списков.
13. Хеш по сумме букв имени по модулю `k`, на базе массива списков.
14. Хеш по сумме цифр телефона по модулю `k`, на базе массива списков.
15–20. Те же 6 хеш-функций (первые/последние `k` букв/цифр, сумма по модулю `k`), но на базе **динамического вектора векторов** длины `m`.
21–26. Те же 6 хеш-функций на базе **B-дерева** основания `m`.
27–32. Те же 6 хеш-функций на базе **AVL-дерева**.
33–38. Те же 6 хеш-функций на базе **красно-чёрного дерева**.
