# Семестр 4 — Задание 5. Мини-СУБД: `insert`/`select`/`delete`, `order by` (v3)

Развитие [Задания 4](../Tasks04/TASK.md): полноценные команды `insert`, `select ... order by`, `delete`, разделённые `;`.

## Требования к программам

1. Дополнительно к структурам из [Задания 4](../Tasks04/TASK.md):
   - Новый enum `command_type`: `none, quit, insert, select, del`.
   - `command` дополняется полями `command_type type`, `ordering order_by[3]` (поля сортировки вывода).
   - Пример разбора: `"select name, group where phone = 1234567 and name like St% and group <> 209 order by group, name;"` → `type=select, name="St%", c_name=like, phone=1234567, c_phone=eq, group=209, c_group=ne, op=land, order={name,group,none}, order_by={group,name,none}`.
2. Задача программы:
   - Построить двунаправленный список `record`, считать из файла.
   - Считывать команды по одной со стандартного ввода до конца потока **или** до команды `quit;`.
   - Выполнять команду над списком, выводя найденные элементы для `select`.
3. Разделитель команд — `;`; разделители аргументов — пробел, `\t`, `\n`. Команды:
   - `quit;` — завершить работу.
   - `insert (<name>, <phone>, <group>);` — добавить запись с указанными полями в список.
   - `select <условия на выводимые поля> [where <условие поиска>] [order by <условия сортировки>];` — вывести записи, удовлетворяющие `where` (по умолчанию — все), в заданном виде и порядке сортировки (по умолчанию — без сортировки).
   - `delete [where <условие поиска>];` — удалить записи, удовлетворяющие `where` (без `where` — удаляется весь список).
4. `<условия сортировки>` — список полей через запятую без повторений; сортировка по первому полю, при равенстве — по следующему, и т.д. (например `order by name, phone`).
5. `<условия на выводимые поля>` и `<условие поиска>` — как в [Задании 4](../Tasks04/TASK.md).
6. Примеры команд: `insert (Student, 1234567, 209);`, `select * where phone >= 1234567 and name like St% order by group;`, `delete where name = Student;`, `select * order by name, phone, group;`.
7. Аргументы командной строки и формат вывода в `main` — как в [Задании 3](../Tasks03/TASK.md).

## Задачи

Написать программу мини-СУБД, поддерживающую `insert`/`select`/`delete`/`quit` над списком `record`. Задание оценивается по качеству системы классов и по реализации:

1. `order by <условия сортировки>` в команде `select`.
2. Команда `insert`.
3. Команда `delete`.
4. Поддержка всех перечисленных команд в едином потоке запросов.
