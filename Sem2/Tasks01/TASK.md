# Семестр 2 — Задание 1. Построчный поиск по шаблону (упрощённые регулярные выражения)

## Требования к программам

1. Задачи оцениваются независимо в двух группах: задачи 1–5 и задачи 6–10.
2. Программа должна получать все параметры в качестве аргументов командной строки.
3. Аргументы командной строки для задач 1–2, 6–10: `fin` – имя входного файла, `fout` – имя выходного файла, `s` – строка. Например, `./a01.out a.txt b.txt "abcd"`.
4. Аргументы командной строки для задач 3–5: `fin`, `fout`, `s` – строка, `t` – строка. Например, `./a05.out a.txt b.txt "abcd" "(){}[] ;+-*/="`.
5. Результатом работы каждой функции является измененный файл (не выводится в `main`) и возвращаемое значение (выводится в `main`).
6. Вывод результата работы функции в функции `main`:

   ```c
   printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
           argv[0], task, res, t);
   ```

   где `res` – результат работы функции, `t` – время работы. Формат должен соблюдаться в точности для автоматизации тестов.

7. Возвращаемым значением всех функций является тип данных, объявленный, например, в файле `io_status.h`:

   ```c
   typedef enum io_status_ {
     SUCCESS,       /* все хорошо */
     ERROR_OPEN,    /* ошибка открытия файла */
     ERROR_READ,    /* ошибка чтения файла */
     ERROR_PATTERN, /* неверная строка поиска */
   } io_status;
   ```

8. Пример функции `main` для задачи 1:

   ```c
   #include <stdio.h>
   #include <time.h>
   #include "io_status.h"

   io_status task1 (const char *a, const char *b, const char *s, int *r);

   int main (int argc, char *argv[])
   {
     int task = 1;
     io_status ret;
     const char * f_in = 0;
     const char * f_out = 0;
     const char * s = 0;
     int res = 0;
     double t;

     if (argc != 4)
       {
         printf("Usage: %s <f_in> <f_out> <s>\n", argv[0]);
         return 1;
       }

     f_in = argv[1];
     f_out = argv[2];
     s = argv[3];
     t = clock();
     ret = task1 (f_in, f_out, s, &res);
     t = (clock() - t) / CLOCKS_PER_SEC;

     switch (ret)
       {
       case SUCCESS:
         printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
                  argv[0], task, res, t);
         break;
       case ERROR_OPEN:
         printf ("Can not open %s\n", f_out);
         return 1;
       case ERROR_READ:
         printf ("Can not read %s\n", f_out);
         return 2;
       case ERROR_PATTERN:
         printf ("Error in pattern %s\n", s);
         return 3;
       }
     return 0;
   }
   ```

## Задачи

Во всех задачах: функция получает имя `a` входного текстового файла неизвестной длины, имя `b` выходного файла, шаблон `s` (и вспомогательную строку `t`, где указано), адрес счётчика `r`. В файл `b` выводятся те строки файла `a`, которые совпадают с шаблоном `s`. Функция возвращает в `r` количество выведенных строк, а как результат — `io_status` (успех/причина неудачи).

1. Шаблон `s` соответствует **вхождению** строки `s` в строку файла; символ `^` в начале `s` означает начало строки (эскейпится как `\^`), `\` эскейпит сам себя.
2. То же самое, но `$` в конце `s` означает конец строки (аналогичный эскейпинг).
3. Шаблон `s` — одно слово (без пробельных символов, разделитель задаётся строкой `t`); последовательность `\<` в начале `s` означает начало слова в строке файла.
4. То же самое, но `\>` в конце `s` означает конец слова.
5. Строки файла, имеющие общее слово со строкой `s` (слово — последовательность без пробельных символов из `t`).
6. Шаблон `s`, где `.` соответствует любому символу строки (эскейпинг `\.`, `\\`).
7. Шаблон `s`, где `?` означает "предыдущий символ 0 или 1 раз" (несколько `?` подряд эквивалентны одному; эскейпинг `\?`, `\\`).
8. То же самое, но `+` означает "предыдущий символ 1 или более раз".
9. То же самое, но `*` означает "предыдущий символ 0 или более раз".
10. Шаблон `s`, где `[n-m]` соответствует любому символу с кодом в диапазоне `n...m` (эскейпинг `\[`, `\]`, `\\`); непарные `[`/`]` — ошибка ввода.
