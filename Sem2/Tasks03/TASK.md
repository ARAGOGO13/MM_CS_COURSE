# Семестр 2 — Задание 3. Сортировки массива строк с функцией сравнения

## Требования к программам

1. Программа должна получать все параметры в качестве аргументов командной строки.
2. Задачи оцениваются независимо в трех группах: задачи 1–3, задачи 4–7 и задачи 8–10.
3. Аргументы командной строки для задач 4–10: `c` – задает номер функции сравнения, `n` – размерность массива, `p` – количество выводимых значений, `filename` – имя файла, откуда прочитать массив. Например, `./a04.out 1 4 4 a.txt`.
4. Аргументы командной строки для задач 1 и 3: добавляется дополнительный первый аргумент `x` (строка), остальные — как в задачах 4–10.
5. Аргументы командной строки для задачи 2: удвоенный "комплект" для массивов `a` и `b`, плюс общий `c`. Например, `./a02.out 1 4 4 a.txt 6 6 b.txt`.
6. Ввод массива должен быть оформлен в виде подпрограммы. В файле находится массив строк, разделённых символом новой строки (сам не входит в строку). Программа должна выводить сообщение об ошибке при проблемах чтения.
7. Задание функции сравнения элементов. В программе должны быть функции сравнения: `c=1` — лексикографически по возрастанию, `c=2` — лексикографически по убыванию, `c=3` — по возрастанию длины строки, `c=4` — по убыванию длины строки. В зависимости от `c` передаётся указатель на соответствующую функцию. Функция сравнения `X` и `Y` возвращает число `<0` при $X \prec Y$, `>0` при $X \succ Y$, `0` при $X \simeq Y$.
8. Решение задачи должно быть оформлено в виде подпрограммы, получающей массив и его длину (и доп. аргументы); дополнительную информацию извне получать и выводить на экран запрещается.
9. Программа должна содержать подпрограмму `print_array` для вывода массива длины не более `p`, по одному элементу на строку.
10. Программа должна содержать функцию, получающую массив `a[n]` строк, число `n`, указатель на функцию `cmp`, и возвращающую количество элементов, строго меньших предыдущего (`diff`).
11. Вывод результата в `main` для задач 4–10:

    ```c
    printf ("New array:\n");
    print_array (a, n, p);
    printf ("%s : Task = %d Diff = %d Elapsed = %.2f\n",
            argv[0], task, diff, t);
    ```

12. Для задачи 3 — формат с `Res` вместо `Diff`; для задачи 2 — массив `c` длины `n+m` с `Diff`; для задачи 1 — только `Res` без массива.

## Задачи

Задачи полностью аналогичны [Sem1/Tasks09](../../Sem1/Tasks09/TASK.md) (те же алгоритмы поиска/слияния/сортировки с компаратором `cmp`), но элементами массива являются **строки**, а не вещественные числа:

1. Двоичный поиск места вставки строки `x` в массив строк, упорядоченный относительно `cmp`. Не более $\log_2 n + O(1)$ сравнений и пересылок.
2. Слияние двух массивов строк `a[n]`, `b[m]`, упорядоченных относительно `cmp`, в массив `c[n+m]` за $n+m+O(1)$ сравнений и пересылок.
3. Разбиение массива строк на элементы `≺ x` и `≻ x` (двусторонний линейный поиск и обмен). Не более $n + O(1)$ сравнений, $3n/2 + O(1)$ пересылок.
4. Пузырьковая сортировка массива строк относительно `cmp`.
5. Сортировка выбором минимума массива строк относительно `cmp`.
6. Сортировка линейными вставками массива строк относительно `cmp`.
7. Сортировка двоичными вставками массива строк относительно `cmp` (алгоритм задачи 1).
8. Сортировка слиянием ("сортировка Неймана") массива строк со вспомогательным массивом `b[n]`, относительно `cmp`.
9. Быстрая сортировка (quicksort) массива строк относительно `cmp`.
10. Пирамидальная сортировка (heapsort) массива строк относительно `cmp`.

Оценки числа сравнений/пересылок в худшем случае — те же, что и в [Sem1/Tasks08](../../Sem1/Tasks08/TASK.md).
