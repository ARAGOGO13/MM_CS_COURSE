# Семестр 3 — Задание 10. Шаблонное бинарное дерево: `tree<student>` и `tree<list2>`

## Требования к программам

1. В программе реализованы следующие структуры данных:
   - **Контейнер `student`** (как в предыдущих заданиях): `print`, `read`, `get_length () const { return (name != nullptr ? 1 : 0); }`, `operator<`.
   - **Двунаправленный список `list2`** (как в [Задании 5](../Tasks05/TASK.md)) с добавленными статическими лимитами:

     ```cpp
     class list2
       {
       private:
         list2_node * head = nullptr;
         static int m; // лимит чтения
         static int r; // лимит печати
       public:
         ...
         static void set_m (int m) { list2::m = m; }
         static void set_r (int r) { list2::r = r; }
         void print (FILE *fp = stdout) const;      // не более r элементов, если r > 0
         io_status read (FILE * fp = stdin);         // не более m элементов, если m > 0
         int get_length () const;
         int operator< (const list2& b) const;
         // пустой список < любого непустого; иначе сравниваются первые элементы
       };
     ```

   - **Шаблонное бинарное дерево** `tree<T>` / `tree_node<T>` (упорядоченное по `T::operator<`), параметризуемое типом `T` (`student` или `list2`):

     ```cpp
     template <class T> class tree;
     template <class T>
     class tree_node : public T
       {
       private:
         tree_node * left = nullptr;
         tree_node * right = nullptr;
       public:
         ...
         friend class tree<T>;
       };
     template <class T>
     class tree
       {
       private:
         tree_node<T> * root = nullptr;
       public:
         ...
         io_status read (FILE * fp = stdin, unsigned int max_read = -1);
         void print (unsigned int r = 10, FILE *fp = stdout) const;
       };
     ```

2. Все функции задания — методы класса `tree`; каждая вызывается **дважды**: для `tree<student>` и для `tree<list2>`.
3. Программа должна получать все параметры в качестве аргументов командной строки: `r` – макс. количество выводимых уровней (оно же — макс. число выводимых элементов списка), `filename`, `m` – количество элементов списка для заполнения `tree<list2>`. Например, `./a.out 4 a.txt 100`.
4. Класс `tree` должен содержать функцию ввода дерева из файла: строки вида `Слово-i Целое-число-i`; первый элемент — в корень, следующие — в левое/правое поддерево по `operator<` элемента дерева. Никакие другие функции, кроме чтения, не используют упорядоченность.
5. Дерево не изменяется функциями задач, кроме последней — поэтому вся работа (для обоих типов деревьев по очереди) идёт в одной `main`:

   ```cpp
   int list2::m = 0;
   int list2::r = 0;

   int main (int argc, char *argv[])
   {
     // ... ввод аргументов ...
     list2::set_m (m);
     list2::set_r (r);
     tree<student> *a = new tree<student>;
     // работа с деревом tree<student>: чтение, вывод, вызов задач 1-6, вывод результата
     delete a;
     tree<list2> *b = new tree<list2>;
     // то же самое для tree<list2>
     delete b;
     return 0;
   }
   ```

6. Вывод результата в `main`:

   ```cpp
   printf ("%s : Task = %d M = %d Result = %d Elapsed = %.2f\n",
           argv[0], m, task, res, t);
   ```

## Задачи

Подпрограмма — член класса `tree<T>` (вызывается для `T = student` и `T = list2`):

1. Возвращает количество элементов типа `T` в концевых узлах дерева.
2. Возвращает максимальное число элементов типа `T` в одной ветви дерева.
3. Возвращает максимальное количество элементов типа `T` в одном уровне дерева.
4. Возвращает максимальную по модулю разность между количеством элементов типа `T` в левом и правом поддереве узла (по всем узлам).
5. Возвращает количество элементов типа `T` в узлах, имеющих ровно 1 потомка.
6. Удаляет все узлы (вместе со всеми потомками), содержащие элемент типа `T` с наименьшим значением `value`/длины среди всех узлов дерева. Возвращает количество удалённых узлов (без учёта потомков).
