# Семестр 3 — Задание 1. Массив объектов `data`: поиск, слияние, сортировки

## Требования к программам

1. Программа работает с массивом объектов класса `data`:

   ```cpp
   class data
   {
      private:
        int m = 0;        // number of actual used elements in a[]
        double a[M] = {}; // array, length is defined in command line -DM=10
        static int p;     // number of printed elements in a[]
      public:
        data () = default;
       ~data () = default;
        int get_m () const { return m; }
        int get_M () const { return (m < M ? m : M); } // min{m,M}
        const double * get_a () { return a; }
        static int get_p () { return p; }
        static void set_p (int q) { p = q; }
        data (const data &x) = default;
        data& operator= (const data&) = default;
        int operator< (const data& x) const { return m < x.m; }
        int operator> (const data& x) const { return m > x.m; }
        // Печать: m a[0] a[1] ... a[m-1] (не более p элементов)
        void print (FILE * fp = stdout) const;
        // Чтение строки: m a[0] a[1] ... a[m-1]
        int read (FILE *fp = stdin);
        // Инициализация по формуле s для i-го элемента
        void init (int s, int n, int i);
   };
   ```

   В класс можно добавлять другие функции, если это требуется для решения задачи.

2. Параметр `M` указывается при компиляции всех файлов программы: `g++ -DM=10 имя_файла.cpp`.
3. Программа должна получать все параметры в качестве аргументов командной строки.
4. Аргументы командной строки для задач 4–10: `n` – размерность массива, `p` – количество выводимых значений, `s` – номер формулы инициализации (0 = файл), `filename`. Например, `./a04.out 4 4 0 a.txt` или `./a04.out 1000000 6 1`.
5. Аргументы командной строки для задачи 1: доп. первый аргумент `x` (имя файла с одним объектом `data`), затем `n`, `p`, `s`, `filename`.
6. Аргументы командной строки для задачи 3: доп. первый аргумент `m` (индекс элемента массива, $m=0,\ldots,n-1$), затем `n`, `p`, `s`, `filename`.
7. Аргументы командной строки для задачи 2: удвоенный "комплект" для массивов `a` и `b`.
8. Ввод массива должен быть оформлен в виде подпрограммы, находящейся в отдельном файле.
9. Ввод массива из файла: каждая строка — `mi a[0] a[1] ... a[mi-1]`; если $m_i > M$, то $m_i = M$ и лишние элементы строки пропускаются.
10. Ввод массива по формуле: $a_i = \{m = f(s,n,i),\ a[j]=0,\ j=0,\ldots,\min\{m,M\}-1\}$, где

    $$
    f(s,n,i) = \begin{cases}
      i & s=1 \\
      n-i & s=2 \\
      i/2 & s=3 \\
      n-i/2 & s=4
    \end{cases}
    $$

    Функция `int f (int s, int n, int i);` работает только с целыми числами; инициализация по формуле оформляется как метод класса `data`.

11. Решение задачи должно быть оформлено в виде подпрограммы в отдельном файле, получающей массив и его длину (и доп. аргументы); дополнительную информацию извне (глобальные переменные, include и т.п.) получать запрещается.
12. Программа должна содержать подпрограмму вывода на экран массива длины не более `p`, по одному элементу на строку.
13. Программа должна содержать подпрограмму для вычисления количества элементов массива, меньших предыдущего элемента (`diff`).
14. Вывод результата в `main`:

    ```cpp
    printf ("%s : Task = %d Diff = %d Elapsed = %.2f\n",
            argv[0], task, diff, t);
    ```

## Задачи

Те же 10 алгоритмов, что и в [Sem1/Tasks08](../../Sem1/Tasks08/TASK.md), но элементами массива являются объекты `data` (сравниваемые по полю `m` через `operator<`/`operator>`):

1. Двоичный поиск места вставки объекта `x` в неубывающий массив `a[n]` (`i` такое, что $a[i] \le x \le a[i+1]$). Не более $\log_2 n + O(1)$ сравнений и перестановок.
2. Слияние двух неубывающих массивов `a[n]`, `b[m]` в неубывающий массив `c[n+m]` за $n+m+O(1)$ сравнений и пересылок.
3. Разбиение массива относительно элемента `a[m]` (двусторонний линейный поиск и обмен, с отслеживанием перемещения индекса `m`). Не более $n/2 + O(1)$ сравнений, $3n/2 + O(1)$ пересылок. Возвращает новую позицию элемента `a[m]`.
4. Пузырьковая сортировка. Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ пересылок.
5. Сортировка выбором минимума. Не более $n^2/2 + O(n)$ сравнений, $3n + O(1)$ пересылок.
6. Сортировка линейными вставками. Не более $n^2/2 + O(n)$ сравнений и пересылок.
7. Сортировка двоичными вставками (алгоритм задачи 1). Не более $n\log_2 n + O(n)$ сравнений, $n^2/2 + O(n)$ пересылок.
8. Сортировка слиянием ("сортировка Неймана") со вспомогательным массивом `b[n]`. Не более $n\log_2 n + O(n)$ сравнений и пересылок.
9. Быстрая сортировка (quicksort), использующая алгоритм задачи 3 для разбиения. Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ пересылок.
10. Пирамидальная сортировка (heapsort). Не более $4n\log_2 n + O(n)$ сравнений, $2n\log_2 n + O(n)$ пересылок.
