# Семестр 3 — Задание 2. Массив объектов `student`: поиск, слияние, сортировки

## Требования к программам

1. Программа работает с массивом объектов класса `student` (поля `name` — динамическая C-строка, `value` — целое число; полный набор конструкторов копирования/перемещения, `operator<`/`operator>`/`operator==`, лексикографическое сравнение по `name`, затем по `value`):

   ```cpp
   enum class io_status { success, eof, format, memory };

   class student
   {
     private:
       char * name = nullptr;
       int    value = 0;
     public:
       student () = default;
       student (const student& x);       // глубокое копирование
       student (student&& x);            // перемещение
       ~student ();
       student& operator= (const student& x);
       student& operator= (student&& x);
       void print () const;              // "%s %d\n", name, value
       io_status read (FILE * fp);       // "%s%d" -> name, value
       int operator> (const student& x) const;
       int operator< (const student& x) const;
       int operator== (const student& x) const;
     private:
       io_status init (const char * n, int v);
       void erase ();
       int cmp (const student& x) const; // по name (strcmp), затем по value
   };
   ```

2. Программа должна получать все параметры в качестве аргументов командной строки.
3. Аргументы командной строки для задач 4–10: `n`, `p`, `s` (0 = файл), `filename`. Например, `./a04.out 4 4 0 a.txt` или `./a04.out 1000000 6 1`.
4. Аргументы командной строки для задачи 1: доп. аргумент `x` — объект `student`, передаваемый как пара (строка, целое число), затем `n`, `p`, `s`, `filename`.
5. Аргументы командной строки для задачи 3: доп. аргумент `m` (индекс элемента, $m=0,\ldots,n-1$), затем `n`, `p`, `s`, `filename`.
6. Аргументы командной строки для задачи 2: удвоенный "комплект" для массивов `a` и `b`.
7. Ввод массива должен быть оформлен в виде подпрограммы в отдельном файле.
8. Ввод массива из файла: строки вида `Слово-i Целое-число-i`, слово — алфавитно-цифровая последовательность без пробелов.
9. Ввод массива по формуле: $a_i = \{\text{"Student"},\ f(s,n,i)\}$, где

   $$
   f(s,n,i) = \begin{cases}
     i & s=1 \\
     n-i & s=2 \\
     i/2 & s=3 \\
     n-i/2 & s=4
   \end{cases}
   $$

   Инициализация по формуле оформляется как метод класса `student`.
10. Решение задачи должно быть оформлено в виде подпрограммы в отдельном файле, получающей массив и его длину (и доп. аргументы); дополнительную информацию извне получать запрещается.
11. Программа должна содержать подпрограмму вывода на экран массива длины не более `p`.
12. Программа должна содержать подпрограмму для вычисления количества элементов массива, меньших предыдущего элемента (`diff`).
13. Вывод результата в `main`:

    ```cpp
    printf ("%s : Task = %d Diff = %d Elapsed = %.2f\n",
            argv[0], task, diff, t);
    ```

## Задачи

Те же 10 алгоритмов, что и в [Sem3/Tasks01](../Tasks01/TASK.md) и [Sem1/Tasks08](../../Sem1/Tasks08/TASK.md), но элементами массива являются объекты `student` (лексикографическое сравнение по `name`, затем по `value`):

1. Двоичный поиск места вставки объекта `x` в неубывающий массив `a[n]`. Не более $\log_2 n + O(1)$ сравнений и перестановок.
2. Слияние двух неубывающих массивов `a[n]`, `b[m]` в неубывающий массив `c[n+m]` за $n+m+O(1)$ сравнений и пересылок.
3. Разбиение массива относительно элемента `a[m]` (двусторонний линейный поиск и обмен, с отслеживанием индекса `m`). Не более $n/2+O(1)$ сравнений, $3n/2+O(1)$ пересылок.
4. Пузырьковая сортировка. Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ пересылок.
5. Сортировка выбором минимума. Не более $n^2/2 + O(n)$ сравнений, $3n + O(1)$ пересылок.
6. Сортировка линейными вставками. Не более $n^2/2 + O(n)$ сравнений и пересылок.
7. Сортировка двоичными вставками (алгоритм задачи 1). Не более $n\log_2 n + O(n)$ сравнений, $n^2/2 + O(n)$ пересылок.
8. Сортировка слиянием ("сортировка Неймана") со вспомогательным массивом `b[n]`. Не более $n\log_2 n + O(n)$ сравнений и пересылок.
9. Быстрая сортировка (quicksort). Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ пересылок.
10. Пирамидальная сортировка (heapsort). Не более $4n\log_2 n + O(n)$ сравнений, $2n\log_2 n + O(n)$ пересылок.
