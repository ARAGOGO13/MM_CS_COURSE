# Семестр 3 — Задание 6. Сортировки однонаправленного списка объектов `student` (класс `list`)

## Требования к программам

1. Программа работает с однонаправленным списком объектов `student`, обёрнутым в класс `list`:

   ```cpp
   class student { /* ... как в Задании 4 ... */ };
   class list_node : public student
     {
     private:
       list_node * next = nullptr;
     public:
       ...
       friend class list;
     };
   class list
     {
     private:
       list_node * head = nullptr;
     public:
       ...
       int read (FILE *fp = stdin, unsigned int max_read = -1);
       void print (unsigned int r = 10, FILE *fp = stdout);
       int get_less_than_previous ();
     };
   ```

   Все функции в задании — методы класса `list`.

2. Программа должна получать все параметры в качестве аргументов командной строки: `r` – количество выводимых значений, `filename` – имя файла со списком. Например, `./a.out 4 a.txt`.
3. Задачи оцениваются независимо в двух группах: задачи 1–3 и задачи 4–5.
4. Класс `list` должен содержать функцию ввода списка из файла (формат — как в [Задании 4](../Tasks04/TASK.md), концом ввода считается конец файла или `max_read`).
5. Решение задачи должно быть оформлено в виде подпрограммы в отдельном файле; дополнительную информацию извне получать запрещается.
6. Класс `list` должен содержать подпрограмму `print` вывода на экран списка длины не более `r`.
7. Класс `list` должен содержать функцию вычисления количества элементов, меньших предыдущего (`get_less_than_previous`), результат которой выводится для результирующего списка.
8. Вывод результата в `main`:

   ```cpp
   printf ("%s : Task = %d Diff = %d Elapsed = %.2f\n",
           argv[0], task, diff, t);
   ```

## Задачи

Подпрограмма — член класса `list`, реализующая сортировку списка **перестановкой ссылок** (не значений), с построением нового списка из старого:

1. **Пузырьковая сортировка**: находить первый элемент, больший следующего, менять местами, продолжать до конца; наибольший элемент оказывается последним — переносить его в начало нового списка, укорачивая исходный на 1; повторять, пока исходный список не опустеет. Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ перестановок ссылок.
2. **Сортировка нахождением минимума**: находить минимальный элемент оставшегося списка, переносить в конец нового списка. Не более $n^2/2 + O(n)$ сравнений, $n + O(1)$ перестановок ссылок.
3. **Сортировка линейной вставкой**: переносить элементы исходного списка по одному, находя линейным поиском место вставки в уже отсортированный новый список. Не более $n^2/2 + O(n)$ сравнений, $n + O(1)$ перестановок ссылок.
4. **Сортировка слиянием (сортировка Неймана)**: отрезать от исходного списка по две группы (изначально по одному элементу), сливать их в упорядоченный список слиянием ссылок, добавлять результат в конец нового списка; когда исходный список опустеет — повторить процесс укрупнения групп (по 2, затем по 4 и т.д.), пока весь список не окажется одной упорядоченной группой. Не более $n\log_2 n + O(n)$ сравнений и перестановок ссылок.
5. **Быстрая сортировка (quicksort)**: за один проход найти элемент из середины списка (два указателя `M`, `T`: `T` продвигается на 2 позиции, `M` — на 1), вырезать его; за ещё один проход разложить остальные элементы на список `L` (меньшие), `U` (большие), `M` (равные, добавляются в конец списка `M`); рекурсивно отсортировать `L` и `U` (сначала более короткий — для уменьшения глубины рекурсии), затем склеить `L + M + U`. Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ перестановок ссылок.
