# Семестр 1 — Задание 9. Сортировки с функцией сравнения

Те же 10 задач, что и в [Задании 8](../Tasks08/TASK.md), но теперь все функции и подпрограммы дополнительно принимают указатель на функцию сравнения `cmp` элементов, и вместо жёсткого сравнения `a[i] < a[i+1]` используется вызов `cmp(a[i+1], a[i])` (или аналогично) — это позволяет сортировать как по возрастанию, так и по убыванию одним и тем же кодом.

## Требования к программам

1. Программа должна получать все параметры в качестве аргументов командной строки.
2. Задачи оцениваются независимо в трех группах: задачи 1–3, задачи 4–7 и задачи 8–10.
3. Аргументы командной строки для задач 4–10:
   1. `c` – задает номер функции, используемой для сравнения значений в массиве,
   2. `n` – размерность массива,
   3. `p` – количество выводимых значений в массиве,
   4. `s` – задает номер формулы для инициализации массива, должен быть равен 0 при вводе массива из файла,
   5. `filename` – имя файла, откуда надо прочитать массив. Этот аргумент отсутствует, если `s != 0`.

   Например, `./a04.out 1 4 4 0 a.txt` — массив длины 4 из файла, выводить не более 4 элементов, функция сравнения 1.

4. Аргументы командной строки для задач 1 и 3: добавляется дополнительный первый аргумент `x` (вещественное число), остальные — как в задачах 4–10 (с `c`).
5. Аргументы командной строки для задачи 2: удвоенный "комплект" аргументов для массивов `a` и `b`, плюс общий аргумент `c`.
6. Ввод массива по формуле $a_i = f(s,n,i)$ — та же таблица, что в [Задании 8](../Tasks08/TASK.md).
7. Задание функции сравнения элементов. В программе должно быть две функции сравнения, обеспечивающих сортировку по возрастанию и по убыванию. При `c = 1` передаётся указатель на первую функцию, при `c = 2` — на вторую. Функция сравнения `X` и `Y` возвращает число `< 0`, если $X \prec Y$; `> 0`, если $X \succ Y$; `0`, если $X \simeq Y$.
8. Решение задачи должно быть оформлено в виде подпрограммы, получающей массив, его длину и указатель на функцию сравнения (и доп. аргументы); дополнительную информацию извне получать и выводить на экран запрещается.
9. Программа должна содержать подпрограмму `print_array` для вывода массива длины не более `p`.
10. Программа должна содержать функцию, получающую массив `a[n]`, число `n`, указатель на функцию `cmp`, и возвращающую количество элементов, строго меньших предыдущего элемента (для проверки, `diff`).
11. Форматы вывода в `main` — как в [Задании 8](../Tasks08/TASK.md), пп. 13–16.

## Задачи

Все задачи 1–10 совпадают по алгоритмам с [Заданием 8](../Tasks08/TASK.md), но сравнение элементов `a[i]` и `a[j]` производится через вызов переданной функции `cmp` вместо встроенных операторов `<`, `>`:

1. Двоичный поиск места вставки `x` в массив, упорядоченный относительно `cmp`.
2. Слияние двух массивов `a[n]`, `b[m]`, упорядоченных относительно `cmp`, в массив `c[n+m]`.
3. Разбиение массива на элементы $\prec x$ и элементы $\succ x$ (двусторонний линейный поиск и обмен).
4. Пузырьковая сортировка относительно `cmp`.
5. Сортировка выбором минимума относительно `cmp`.
6. Сортировка линейными вставками относительно `cmp`.
7. Сортировка двоичными вставками относительно `cmp` (использует алгоритм задачи 1).
8. Сортировка слиянием ("сортировка Неймана") с вспомогательным массивом `b[n]`, относительно `cmp`.
9. Быстрая сортировка (quicksort) относительно `cmp`.
10. Пирамидальная сортировка (heapsort) относительно `cmp`.

Оценки числа сравнений/пересылок в худшем случае — те же, что и в [Задании 8](../Tasks08/TASK.md).
