# Семестр 1 — Задание 8. Сортировки массивов

## Требования к программам

1. Программа должна получать все параметры в качестве аргументов командной строки.
2. Задачи оцениваются независимо в трех группах: задачи 1–3, задачи 4–7 и задачи 8–10.
3. Аргументы командной строки для задач 4–10:
   1. `n` – размерность массива,
   2. `p` – количество выводимых значений в массиве,
   3. `s` – задает номер формулы для инициализации массива, должен быть равен 0 при вводе массива из файла,
   4. `filename` – имя файла, откуда надо прочитать массив. Этот аргумент отсутствует, если `s != 0`.

4. Аргументы командной строки для задач 1 и 3: добавляется дополнительный первый аргумент `x` (вещественное число), остальные — как в задачах 4–10.
5. Аргументы командной строки для задачи 2: удвоенный "комплект" аргументов из задач 4–10 для двух массивов `a` и `b`.
6. Ввод массива должен быть оформлен в виде подпрограммы; ввод из файла или по формуле $a_i = f(s,n,i)$:

   $$
   f(s,n,i) = \begin{cases}
     i & s = 1 \\
     n - i & s = 2 \\
     i / 2.0 & s = 3 \\
     n - i/2.0 & s = 4 \\
     2.0 \cdot i & s = 5 \\
     n - 2.0 \cdot i & s = 6
   \end{cases}
   $$

7. Решение задачи должно быть оформлено в виде подпрограммы, получающей массив и его длину (и доп. аргументы); дополнительную информацию извне получать и выводить на экран запрещается.
8. Программа должна содержать подпрограмму `print_array` вывода на экран массива длины не более `p`, по одному элементу на строку.
9. Программа должна содержать функцию, получающую массив `a[n]`, число `n`, и возвращающую количество элементов массива, строго меньших предыдущего элемента (используется для проверки правильности сортировки, `diff`).
10. Вывод результата работы в функции `main` для задач 4–10:

    ```c
    printf ("New array:\n");
    print_array (a, n, p);
    printf ("%s : Task = %d Diff = %d Elapsed = %.2f\n",
            argv[0], task, diff, t);
    ```

11. Для задачи 3 — формат с `Res` вместо `Diff`; для задачи 2 — вывод массива `c` длины `n+m` с `Diff`; для задачи 1 — вывод только `Res` без массива (см. код).

## Задачи

Основная программа заполняет массив данными, выводит его на экран, вызывает подпрограмму и выводит результат работы.

1. Двоичный поиск места вставки `x` в неубывающий массив `a[n]` (минимальное `i` такое, что $x \le a[i]$, иначе $i = n$): взять границы `0` и `n`; пока границы не совпадут, сравнивать `x` с `a[s]` (`s` — середина границ) и сдвигать соответствующую границу. Не более $\log_2 n + O(1)$ сравнений и пересылок.
2. Слияние двух неубывающих массивов `a[n]`, `b[m]` в неубывающий массив `c[n+m]` за $n+m+O(1)$ сравнений и пересылок (аналогично Tasks06 №9, но без выбрасывания повторов).
3. Разбиение массива на элементы `< x` и элементы `> x` (двусторонний линейный поиск и обмен встречных элементов, пока границы не пересекутся). Не более $n + O(1)$ сравнений и $3n/2 + O(1)$ пересылок. Возвращает место вставки `x`.
4. Пузырьковая сортировка: последовательными проходами находить наименьшее `i`, для которого `a[i] > a[i+1]`, менять местами, продолжать с `a[i+1]`; каждый проход "выталкивает" наибольший элемент в конец. Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ пересылок.
5. Сортировка выбором минимума: находить минимальный элемент оставшейся части и ставить на своё место. Не более $n^2/2 + O(n)$ сравнений, $3n + O(1)$ пересылок.
6. Сортировка вставками (линейный поиск места вставки). Не более $n^2/2 + O(n)$ сравнений и пересылок.
7. Сортировка вставками с двоичным поиском места вставки (см. задачу 1). Не более $n \log_2 n + O(n)$ сравнений, $n^2/2 + O(n)$ пересылок.
8. Сортировка слиянием ("сортировка Неймана") с вспомогательным массивом `b[n]`: группы по одному элементу сливаются попарно (см. задачу 2), группы укрупняются пока не останется одна. Не более $n \log_2 n + O(n)$ сравнений и пересылок.
9. Быстрая сортировка (quicksort): `x = a[n/2]`, разбиение массива алгоритмом задачи 3, рекурсивная сортировка обеих частей (сначала более короткой — для уменьшения глубины рекурсии, вторая часть — через переход к началу процедуры вместо повторного вызова). Не более $n^2/2 + O(n)$ сравнений, $3n^2/2 + O(n)$ пересылок.
10. Пирамидальная сортировка (heapsort): массив как бинарное дерево (потомки `a[k]` — `a[2k+1]`, `a[2k+2]`); построение упорядоченной пирамиды продвижением каждого элемента по цепочке родителей, затем последовательное извлечение максимума из корня с восстановлением структуры пирамиды. Не более $4n \log_2 n + O(n)$ сравнений, $2n \log_2 n + O(n)$ пересылок.
