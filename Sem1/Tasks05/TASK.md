# Семестр 1 — Задание 5. Битовые последовательности из файла

## Требования к программам

1. Программа должна получать все параметры в качестве аргументов командной строки.
2. Задачи оцениваются независимо в двух группах: задачи 1–4 и задачи 5–8.
3. Аргументы командной строки для задач 1–8:
   1. `name` – имя файла.

   Например, запуск `./a.out a.txt` означает, что имя файла – это `a.txt`.

4. Решение задачи должно быть оформлено в виде функции.
5. В функции, реализующей решение задачи, запрещается что-либо выводить на экран.
6. В программе должна быть функция

   ```c
   int print_bits (const char*);
   ```

   получающая в качестве аргумента имя файла, содержащего последовательность битов неизвестной длины (в файле битовая последовательность задается как последовательность целых чисел, двоичное представление которых **без ведущих 0** образует битовую последовательность), и выводящая эту битовую последовательность на экран в одну строчку и без пробелов. Функция должна возвращать −1, −2 и т.д. (как именованную константу, описанную в файле `io_status.h`), если она не смогла открыть файл, прочитать элемент и т.д.

7. В программе должна быть функция

   ```c
   int print_bits0 (const char*);
   ```

   аналогичная предыдущей, но битовое представление чисел берётся **с ведущими 0**.

8. Функции `print_bits` и `print_bits0` размещаются в отдельном файле с именем `print_bits.c`, а их прототипы – в файле `print_bits.h`. Файл `print_bits.h` включается в файл `print_bits.c` и в файлы, где находятся реализации заданий.

9. Пример файла `print_bits.h`:

   ```c
   int print_bits (const char*);
   int print_bits0 (const char*);
   ```

10. Пример файла `print_bits.c`:

    ```c
    #include <stdio.h>
    #include "io_status.h"
    #include "print_bits.h"
    /* реализация функций */
    ...
    ```

11. Пример файла реализующей задание:

    ```c
    #include <stdio.h>
    #include "io_status.h"
    #include "print_bits.h"
    /* реализация функции, решающей задачу, и функции main */
    ...
    ```

12. Файл `print_bits.c` добавляется в команду компиляции задач. Пример компиляции для задачи 1:

    ```
    gcc task01.c print_bits.c -o a01.out
    ```

13. Вывод результата работы в функции `main` для задач 1, 3, 5, 7 должен производиться по формату:

    ```c
    print_bits (argv[1]);
    printf ("%s : Task = %d Result = %d i = %d j = %d\n",
            argv[0], task, res, i, j);
    ```

14. Вывод результата работы в функции `main` для задач 2, 4, 6, 8 должен производиться по формату:

    ```c
    print_bits0 (argv[1]);
    printf ("%s : Task = %d Result = %d i = %d j = %d\n",
            argv[0], task, res, i, j);
    ```

## Задачи

Во всех задачах: нумерация битов начинается с 0. Функция должна возвращать −1, −2 и т.д. (именованную константу из `io_status.h`), если она не смогла открыть файл, прочитать элемент и т.д., при этом переменные `i`, `j` не изменяются.

1. Функция, получающая имя файла с последовательностью битов (без ведущих 0) и адреса целых `i`, `j`, возвращающая количество элементов последовательности, больших предыдущего элемента. `i` — номер первого такого бита, `j` — номер последнего.
2. То же самое, но битовое представление берётся с ведущими 0, и считается количество элементов, больших **двух следующих** элементов.
3. Функция для последовательности без ведущих 0: количество строгих локальных минимумов ($x_{i-1} > x_i < x_{i+1}$), `i`/`j` — номера первого/последнего минимума.
4. То же самое с ведущими 0, но для строгих локальных максимумов ($x_{i-1} < x_i > x_{i+1}$).
5. Функция для последовательности без ведущих 0: длина максимального участка из единичных битов; `i` — номер бита начала первого такого участка, `j` — начала последнего.
6. То же самое с ведущими 0, но для максимального участка из нулевых битов.
7. Функция для последовательности без ведущих 0: количество участков, составленных из единичных битов; `i`/`j` — номера начала первого/последнего такого участка.
8. То же самое с ведущими 0, но для участков, составленных из нулевых битов.
