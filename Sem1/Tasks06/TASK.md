# Семестр 1 — Задание 6. Массивы: базовые операции

## Требования к программам

1. Программа должна получать все параметры в качестве аргументов командной строки.
2. Задачи оцениваются независимо в двух группах: задачи 1–5 и задачи 6–10.
3. Аргументы командной строки для задач 1–4:
   1. `n` – размерность массива,
   2. `p` – количество выводимых значений в массиве,
   3. `s` – задает номер формулы для инициализации массива, должен быть равен 0 при вводе массива из файла,
   4. `filename` – имя файла, откуда надо прочитать массив. Этот аргумент отсутствует, если `s != 0`.

   Например, запуск `./a01.out 4 4 0 a.txt` означает, что массив длины 4 надо прочитать из файла `a.txt`, и выводить не более 4-х элементов массива, а запуск `./a01.out 1000000 6 1` означает, что массив длины 1000000 надо инициализировать по формуле номер 1, и выводить не более 6-ти элементов массива.

4. Аргументы командной строки для задач 5–8: добавляется дополнительный первый аргумент, остальные аргументы – как в задачах 1–4:
   1. `X` – дополнительный аргумент: `X = k` в задаче 5 (`k` – целое число), `X = x` в задаче 6 (`x` – вещественное число), `X = b` в задачах 7 и 8 (`b` – имя файла),
   2. `n`, `p`, `s`, `filename` – как выше.

5. Аргументы командной строки для задач 9–10: для задания двух входных массивов используется удвоенный "комплект" аргументов из задач 1–4 (для массива `a`: `n, pa, sa, filenamea`; для массива `b`: `m, pb, sb, filenameb`).

   Например, запуск `./a09.out 4 4 0 a.txt 6 6 0 b.txt` читает массив `a` длины 4 из `a.txt` и массив `b` длины 6 из `b.txt`.

6. Ввод массива должен быть оформлен в виде подпрограммы.
7. Ввод массива из файла: в указанном файле находится массив чисел, разделенных пробелами или символом новой строки. Программа должна выводить сообщение об ошибке, если файл не может быть прочитан, содержит меньше данных или данные неверного формата.
8. Ввод массива по формуле. Элемент $a_i$ массива $A$ полагается равным $a_i = f(s, n, i)$, $i = 1, \ldots, n$, где функция `f(s, n, i)` оформляется в виде отдельной подпрограммы:

   $$
   f(s,n,i) = \begin{cases}
     i & s = 1 \\
     n - i & s = 2 \\
     i / 2.0 & s = 3 \\
     n - i/2.0 & s = 4 \\
     2.0 \cdot i & s = 5 \\
     n - 2.0 \cdot i & s = 6
   \end{cases}
   $$

9. Решение задачи должно быть оформлено в виде подпрограммы, получающей в качестве аргументов массив и его длину (в ряде задач также дополнительные аргументы). Получать в этой подпрограмме дополнительную информацию извне, а также выводить что-либо на экран, запрещается.
10. Программа должна содержать подпрограмму `print_array` вывода на экран массива длины не более `p`. Используется для вывода исходного и результирующего массива. Каждый элемент печатается на новой строке.
11. Вывод результата работы в функции `main` для задач 1, 7, 8:

    ```c
    printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
            argv[0], task, res, t);
    ```

12. Вывод результата работы в функции `main` для задач 3–5:

    ```c
    printf ("New array:\n");
    print_array (a, n, p); /* вывод нового состояния массива a */
    printf ("%s : Task = %d Elapsed = %.2f\n",
            argv[0], task, t);
    ```

13. Вывод результата работы в функции `main` для задач 6, 10:

    ```c
    printf ("New array:\n");
    print_array (a, res, p);
    printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
            argv[0], task, res, t);
    ```

14. Вывод результата работы в функции `main` для задачи 9:

    ```c
    printf ("New array:\n");
    print_array (c, res, p); /* вывод массива c длины res */
    printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
            argv[0], task, res, t);
    ```

## Задачи

1. Функция, определяющая, симметричен ли массив вещественных чисел (возвращает 1/0).
2. Подпрограмма, переставляющая элементы массива в обратном порядке не более чем за $3n/2 + O(1)$ перемещений.
3. Подпрограмма, заменяющая каждый элемент массива (кроме первого и последнего) на полусумму соседей.
4. Подпрограмма, циклически сдвигающая элементы массива на одну позицию вправо не более чем за $n + O(1)$ перемещений.
5. Подпрограмма, циклически сдвигающая элементы массива на `k` позиций вправо не более чем за $n + O(1)$ перемещений.
6. Функция, выбрасывающая из массива все элементы, меньшие `x` (сдвигая последующие элементы), не более чем за $n + O(1)$ перемещений; возвращает новую длину массива.
7. Функция, возвращающая количество элементов последовательности из файла `b`, совпадающих с каким-либо элементом массива `a[n]`.
8. Функция, возвращающая количество вхождений подпоследовательности `a[n]` (n подряд идущих совпадающих элементов) в последовательность из файла `b`.
9. Подпрограмма слияния двух строго возрастающих массивов `a[n]`, `b[m]` в строго возрастающий массив `c[n+m]` (с выбрасыванием повторов) не более чем за $2(n+m) + O(1)$ сравнений и $n+m+O(1)$ пересылок. Алгоритм: сравниваем очередные `a[i]` и `b[j]`; если `a[i] < b[j]`, кладём `a[i]` и увеличиваем `i`; если `a[i] > b[j]` — кладём `b[j]` и увеличиваем `j`; при равенстве увеличиваем оба индекса. Возвращает длину массива `c`.
10. Подпрограмма, выбрасывающая из массива `a` все элементы, равные какому-либо элементу массива `b`, не более чем за $n + O(1)$ перемещений; возвращает новую длину `a`.
