# Семестр 1 — Задание 7. Массивы: фильтрация по агрегатам

## Требования к программам

1. Программа должна получать все параметры в качестве аргументов командной строки.
2. Задачи оцениваются независимо в двух группах: задачи 1–4 и задачи 5–8.
3. Аргументы командной строки для задач 1–3:
   1. `n` – размерность массива,
   2. `p` – количество выводимых значений в массиве,
   3. `s` – задает номер формулы для инициализации массива, должен быть равен 0 при вводе массива из файла,
   4. `filename` – имя файла, откуда надо прочитать массив. Этот аргумент отсутствует, если `s != 0`.

   Например, `./a01.out 4 4 0 a.txt` — прочитать массив длины 4 из `a.txt`, выводить не более 4 элементов; `./a01.out 1000000 6 1` — массив длины 1000000 по формуле 1, выводить не более 6 элементов.

4. Аргументы командной строки для задач 4, 5, 8: добавляется дополнительный первый аргумент `X`: `X = k` в задачах 4, 5 (целое число), `X = b` в задаче 8 (имя файла); остальные аргументы — как в задачах 1–4.
5. Аргументы командной строки для задач 6, 7:
   1. `n` – размерность массива,
   2. `p` – количество выводимых значений в массиве,
   3. `filename` – имя файла `b`.

6. Ввод массива должен быть оформлен в виде подпрограммы.
7. Ввод массива из файла: числа разделены пробелами или новой строкой; программа выводит сообщение об ошибке при проблемах чтения.
8. Ввод массива по формуле $a_i = f(s, n, i)$:

   $$
   f(s,n,i) = \begin{cases}
     i & s = 1 \\
     n - i & s = 2 \\
     i / 2.0 & s = 3 \\
     n - i/2.0 & s = 4 \\
     2.0 \cdot i & s = 5 \\
     n - 2.0 \cdot i & s = 6
   \end{cases}
   $$

9. Решение задачи должно быть оформлено в виде подпрограммы, получающей массив и его длину (и дополнительные аргументы); дополнительную информацию извне получать и выводить что-либо на экран запрещается.
10. Программа должна содержать подпрограмму `print_array` вывода на экран массива длины не более `p`, по одному элементу на строку.
11. Вывод результата работы в функции `main` для задач 1–7:

    ```c
    printf ("New array:\n");
    print_array (a, res, p);
    printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
            argv[0], task, res, t);
    ```

12. Вывод результата работы в функции `main` для задачи 8: тот же формат, что и в п. 11.

## Задачи

Во всех задачах основная программа заполняет массив данными, выводит его на экран, вызывает функцию/подпрограмму и выводит результат её работы. Функция возвращает длину получившегося массива.

1. Выбросить из массива все элементы, меньшие среднего арифметического тех элементов, для которых определено среднее геометрическое соседних элементов и которые меньше этого среднего геометрического (не более $2n + O(1)$ пересылок).
2. Выбросить из массива все элементы, меньшие среднего арифметического элементов, находящихся в участках постоянства исходного массива (не более $2n + O(1)$ пересылок).
3. Выбросить из массива все элементы, меньшие среднего арифметического элементов первого самого длинного участка неубывания массива (не более $2n + O(1)$ пересылок).
4. Выбросить из массива (с параметром `k`) все элементы, меньшие максимального элемента во всех участках постоянства длиной не менее `k` (не более $2n + O(1)$ пересылок).
5. Выбросить из массива (с параметром `k`) все элементы, меньшие среднего арифметического элементов участков невозрастания длиной не менее `k` (не более $2n + O(1)$ пересылок).
6. Присвоить `a[i]`, $i = 0, \ldots, n-1$, значение i-го по минимальной величине элемента последовательности из файла `b`, находящегося в участках постоянства. Функция возвращает количество заполненных элементов.
7. То же самое, но `a[i]` получает значение i-го по максимальной величине строгого локального максимума последовательности из файла `b`.
8. Функция, возвращающая количество вхождений подпоследовательности `a[n]` в последовательность из файла `b` (например, для `a = {1,1}` и `b = {1,1,1,1,1}` число вхождений равно 4).
